# Issue Templates

This project includes several issue templates to standardize issue reporting and improve project management.

## Available Templates

The following issue templates are available in the `.github/ISSUE_TEMPLATE` directory:

1. **Bug Report** (`bug-report.md`)
   - For reporting bugs and unexpected behavior
   - Includes fields for steps to reproduce, expected behavior, and actual behavior

2. **Feature Request** (`feature-request.md`)
   - For suggesting new features or enhancements
   - Includes fields for feature description, use cases, and alternatives considered

3. **Documentation Bug** (`docs-bug.md`)
   - For reporting issues with documentation
   - Includes fields for documentation location and suggested improvements

4. **Enhancement Request** (`enhancement-request.md`)
   - For suggesting improvements to existing features
   - Includes fields for current behavior and desired behavior

5. **Failing Test** (`failing-test.md`)
   - For reporting test failures
   - Includes fields for test name, error message, and environment details

6. **Question/Support** (`question-support.md`)
   - For general questions and support requests
   - Includes fields for question details and context

7. **Security Report** (`security-report.md`)
   - For reporting security vulnerabilities
   - Includes fields for vulnerability details and impact

## Configuration

The issue templates are configured in `.github/ISSUE_TEMPLATE/config.yml`, which defines:

- Template selection screen
- External links for additional resources
- Default labels for each template type
